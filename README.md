# GP_2_Scrapping_-_API

### Цель проекта - 
провести аналитическое исследование рынка аренды жилой недвижимости в выбранном городе, чтобы определить, какие характеристики объектов и особенности текстового описания связаны с более высокой стоимостью аренды.

### Данные
Источник данных - сервис Яндекс Недвижимость. https://realty.yandex.ru/moskva/

Сервис представляет из себя агрегатор объявлений о продаже недвижимости в России. В нашем анализе мы рассматривали только регион Москва и близлежаший пригород.

Решение взять именно Москву связано с тем что мы хотим добиться большей однородности данных, если брать данные из разных регионов/рынков мы будем иметь неоднородные данные в силу объективных причин

Другие сервисы которые мы рассматривали при выборе источника - Циан и Домклик. С ними процесс скрапинга информации немного тяжелее и мы выбрали сервис требующий меньше затрат в реализации парсинга данных

Для сбора данных использовалась библиотека selinium, с браузером Chrome и базовыми настройками.

Пайплайн сбора данных: собираем ссылки -> по каждой найденной ссылке достаем необходимые параметры

Всего на сервисе существует порядка 70 тысяч объявлений(как говорит сам сервис), но получить все объявления на одной странице или просто постоянно переходя на новую страницу - невозможно. Так как поисковая строка на сервисе предлагает только 25 страниц выдачи(у конкурентов Циан и Домклик по 50). Мы приняли решение работать с фильтрацией данных по этажам, то есть собирать объявления по недвижимости по отдельным этажам с 1 по 30 брали 25 страниц объявлений по каждому этажу, с 30 по 60 брали по 25 страниц объявлений на диапазон в 5 этажей(31-35,36-40 и т.д.) и на объявления с этажом выше 60 взяли отдельно 25 страниц
