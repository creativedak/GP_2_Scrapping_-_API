# GP_2_Scrapping_-_API

### Цель проекта: 
Провести аналитическое исследование рынка продажи жилой недвижимости в выбранном городе, чтобы определить, какие характеристики объектов и особенности текстового описания связаны с более высокой стоимостью.

### Данные
#### Скрапинг
Источник данных - сервис Яндекс Недвижимость. https://realty.yandex.ru/moskva/

Сервис представляет из себя агрегатор объявлений о продаже недвижимости в России. В нашем анализе мы рассматривали только регион Москва и близлежаший пригород.

Решение взять именно Москву связано с тем что мы хотим добиться большей однородности данных, если брать данные из разных регионов/рынков мы будем иметь неоднородные данные в силу объективных причин

Другие сервисы которые мы рассматривали при выборе источника - Циан и Домклик. С ними процесс скрапинга информации немного тяжелее и мы выбрали сервис требующий меньше затрат в реализации парсинга данных

Для сбора данных использовалась библиотека selinium, с браузером Chrome и базовыми настройками.

Пайплайн сбора данных: собираем ссылки -> по каждой найденной ссылке достаем необходимые параметры

Всего на сервисе существует порядка 70 тысяч объявлений(как говорит сам сервис), но получить все объявления на одной странице или просто постоянно переходя на новую страницу - невозможно. Так как поисковая строка на сервисе предлагает только 25 страниц выдачи(у конкурентов Циан и Домклик по 50, то есть проблема точно такая же). Мы приняли решение работать с фильтрацией данных по этажам, то есть собирать объявления по недвижимости по отдельным этажам:
- с 1 по 30 брали 25 страниц объявлений по каждому этажу,
- с 30 по 60 брали по 25 страниц объявлений на диапазон в 5 этажей(31-35,36-40 и т.д.)
- на объявления с этажом выше 60 взяли отдельно 25 страниц

Таким образом мы соскрапили ссылки и получили порядка 14 тысяч строк, дальше по каждой ссылке скрапили следующие признаки:

- Ссылка - ссылка на объявление на платформе
- Широта - географическая широта объекта
- Долгота - географическая долгота объекта
- Цена - стоимость объекта недвижимости в рублях
- Метро - ближайшая станция метро к объекту недвижимости
- Общая площадь - площадь объекта в квадратных метрах
- Этаж квартиры - этаж на котором находится объект
- Этажность дома - количество этаже в здании объекта
- Потолки - высота потолков объекта недвижимости
- Год - год постройки/сдачи объекта(может быть > 2026) объект может быть еще не сдан
- Описание - описание объекта с объявления
- Имя застройщика - название застройщика, указанное в объявлении(не юридическое название)

#### API
Мы собрали датасет на 14 тысяч объектов с помощью скрапинга и теперь обогатим эти данные через API

Мы берем следующие API сервисы:
##### Walkscore API
Ссылка на сайт - https://www.walkscore.com/professional/. Эндпоинта запроса - https://api.walkscore.com/score

Этот API позволяет получить по координатам точки Walkscore - оценку уровня пешей доступности района, грубо говоря отвечает на вопрос можно ли тут жить и ходить пешком не пользуясь машиной, чем больше показатель тем лучше район для пешеходов. Из этого API мы берем признак:
- оценка пешей доступности - тот самый walkscore

##### Air Quality API Ninjas  
Ссылка на сайт - https://api-ninjas.com/api/airquality. Эндпоинт запроса - https://api.api-ninjas.com/v1/airquality

Этот API позволяет получить по координатам точки качество воздуха в данном месте. Чем выше значение - тем более загрязненный воздух. Из данного API мы берем показатель:
- overall_aqi - Air Quality Index (AQI), это единая шкала для оценки уровня загрязнения воздуха и его влияния на здоровье

##### Geoapify API 
Ссылка на сайт - https://www.geoapify.com/. 

Через это API мы будем получать разные показатели, всего будет три эндпоинта:
- https://api.geoapify.com/v1/geocode/reverse - запрос к данному эндпоинту позволяет получить адрес в международном формате, адрес понадобиться нам, чтобы получить walkscore из другого API
- https://api.geoapify.com/v2/places - запрос к данному эндпоинту позволяет получить места поблизости, такие как кафе, рестораны и т.д. Мы будем считать количество таких мест поблизости 
- https://api.geoapify.com/v1/routing - запрос к этому эндпоинту дает нам маршрут между двумя точками, мы будем использовать его для того, чтобы посчитать время в пути на машине до центра города

Итого, через данный сервис мы получаем следующие признаки:
- адресс - адресс на английском языке, в международном формате
- время до центра в минутах - время в пути на машине до красной площади
- количество кафе в радиусе 1 км - так и есть
- количество ресторанов в радиусе 1 км - так и есть
- количество школ в радиусе 1 км - так и есть
- количество парков в радиусе 1 км - так и есть
- количество супермаркетов в радиусе 1 км - так и есть
- количество общественного транспорта в радиусе 1 км - так и есть

### Ограничения проекта

#### Данные собирались распределённо
В силу большого объема информации, из за тайм слипов(без которых код будет падать) спарсить сначала все 14 тысяч ссылкок, а после по каждой ссылке информацию об объекте представляется довольно сложной задачей, по нашим примерным расчетам сбор ссылок занял бы порядка 12 часов, и примерно столько же получение информации по каждому объекту. Суммарный скрапинг без API занимал бы больше 1 дня. Поэтому мы распределили скрапинг между участниками проекта и делали его на разных устройствах. Для сбора ссылок мы разделились по этажам. После на этапе скрапинга конкретных объявлений мы распределились просто по порядку, каждый член команды взял себе часть ссылок и объявлений. Результаты мы просто смерджили и получили итоговый датасет, файл с логами мы также соединили. Работа с API проводилась точно также, сначала поделили данные на группы, каждый запустил часть на своем устройстве и после мы смерджили данные. Коды с помощью которых происходил merge датасетов представлены в отдельной директории `merge_helpers`

### Feature Engeneering
В проекте мы провели этап работы с признаками, мы создали новые признаки, что то трансформировали, часть признаков мы удалили, часть решили не использовать

Новые признаки:
- Цена за м² тыс - цена квартиры сама по себе сильно зависит от площади, а цена за м² позволяет сравнивать объекты более корректно, в тыс. руб. Тип - `float`
- Возраст дома - год дома менее интерпретируем, а возраст дома проще анализировать и сравнивать. Тип - `float`
- Настроение описания - тональность описания объявления. Тип - `object`
- Уровень качества воздуха - категориальный признак, принимает значения: низкий/средний/высокий. Тип - `category`
- Уровень инфраструктуры - показывает насколько развитая инфраструктура рядом с объектом, этот признак получается с помощью объединения признаков связанных с количество объектов рядом. Тип - `object`
- Уровень этажа - показывает то насколько высоко находится объект недвижимости относительно своего здания. Тип `object`
